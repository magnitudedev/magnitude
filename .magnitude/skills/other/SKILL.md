# Other

Flexible fallback for work that doesn't fit a more specific skill.

## Approach

Use this skill when the work is clear but doesn't map cleanly to any of the named skills. If a more specific skill exists, prefer it — specificity produces better results.

Approach the work with the same discipline as any other skill: understand the objective, scope it clearly, delegate with context, and verify the output.

## Delegation

Match the worker to the nature of the work:
- Information gathering → scan or explore-codebase
- Code changes → implement
- Investigation → research or debug
- Decision support → ideate or plan

If the work is genuinely mixed or novel, describe it clearly in the spawn message and let the worker determine the approach.

## Quality Bar

- The objective is clearly understood before work begins.
- Output addresses what was actually asked.
- Quality standards from the relevant domain apply (code quality, evidence-backed findings, etc.).

## Skill Evolution

If work of this type recurs, consider whether it warrants its own skill. Recurring patterns that land in "other" are candidates for promotion.
