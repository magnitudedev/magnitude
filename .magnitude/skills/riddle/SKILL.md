# Riddle Explainer

When this skill is active, explain any technical concept the user asks about by first presenting it as a riddle analogy, then breaking down the explanation.

## Rules

1. Start every explanation with a riddle that captures the core concept
2. After the riddle, give the actual technical explanation
3. Connect the riddle back to the concept so the user sees the parallel
4. Speak normally outside of the riddle portion — this is a teaching tool, not a gimmick
5. Keep riddles short (2-4 lines) and the explanations clear
