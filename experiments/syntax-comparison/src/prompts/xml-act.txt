You are participating in a syntax comparison experiment.

Produce exactly one turn in current xml-act format.

Grammar:
response  := lenses comms actions turnctl
lenses    := <lenses> lens+ </lenses>
lens      := <lens name="...">...</lens>
comms     := <comms> message* </comms>
message   := <message to="user">...</message>
actions   := <actions> tool* </actions> | <actions/>
turnctl   := <next/> | <yield/>

Required shape:
<lenses>
  <lens name="intent">...</lens>
  <lens name="strategy">...</lens>
</lenses>
<comms>
  <message to="user">...</message>
</comms>
<actions>
  zero or more tool calls
</actions>
<next/> or <yield/>

Rules:
- Start with one <lenses> block containing one or more <lens name="..."> blocks.
- Then include one <comms> block with any user-facing communication.
- Then include one <actions> block with tool calls.
- End with exactly one turn-control tag: <next/> or <yield/>.
- Every tool call MUST include an observe attribute.
  - Use observe="." to request full output.
  - Or use an XPath-style expression to request a subset.
- There is no <inspect> block in this format.
- Progressive disclosure is done using observe="..." on tool calls.
- Use <next/> when you need tool results before continuing.
- Use <yield/> when done or waiting for external input.
- Do not claim file contents, search results, or command output unless already observed in a prior turn.
- Content is literal text; do NOT use HTML entity escaping.
- This is not standard XML parsing mode; do not escape with entities.
- Return only raw XML, no markdown code fences.

Mock tools:

<write path="..." observe=".">content</write>
<edit path="..." observe="."><old>...</old><new>...</new></edit>
<search pattern="..." path="..." observe="."/>

<tree path="..." observe="."/>
