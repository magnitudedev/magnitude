## shell

Execute a shell command.

```xml
<shell>npm test</shell>
```

Returns:

```xml
<shell>
  <stdout>...</stdout>
  <stderr>...</stderr>
  <exitCode>0</exitCode>
</shell>
```

- Commands run with a 30-second timeout
- Output is capped at 10MB
- Returns stdout, stderr, and exit code

## Permissions

Shell commands are classified into three tiers:

- **Read-only** — commands like `ls`, `cat`, `grep`, `find`, `git status`, and `git log` that only read data. These always run freely.
- **Standard** — commands like `npm install`, `npm test`, or `mkdir` that modify files or run processes. These are allowed within the current working directory but **blocked if they write outside it**.
- **Forbidden** — destructive commands like `rm -rf /`, `mkfs`, or `dd` targeting system paths. These are always blocked.

**Non-read-only git commands are always blocked.**

### Working directory restriction

Standard commands are restricted to your project directory. Any command that writes to paths outside the current working directory is blocked (with exceptions for `/tmp` and `/dev/null`).

### Per-agent access

Not all agents have the same shell access:

- **Read-only agents** (scout, researcher, planner) — can only run read-only commands. Standard and forbidden commands are rejected.
- **Write agents** (orchestrator, builder, debugger, reviewer) — can run read-only and standard commands within the working directory. Forbidden commands are always rejected.
