This page covers the practical workflow for collaborating with subagents in the interface.

## How delegated work appears

When the team lead delegates work, you will see brief lifecycle updates in the main conversation.

Subagents also appear as tasks near the composer. Use the task list to switch context and work directly with a subagent.

## Switching context with the task list

A task list above the composer shows:
- the **Main** chat context,
- one task row per active or idle subagent.

Select a task context to choose where your next message goes. Task rows also show live status so you can monitor work without opening details.

 See [Commands](./commands) for shortcut details.

## Messaging a subagent directly

You can message a subagent directly by selecting its task context and sending a message.

Use this for targeted guidance, for example:
- clarifying acceptance criteria,
- requesting a focused check,
- redirecting implementation approach.

If the Main chat context is selected, your message goes to the team lead as usual.

## Viewing details and nested delegation

Open a subagent task context to view that subagent's messages and reasoning stream.

If that subagent delegates further, you can continue drilling into nested activity from its detail view.

## Closing and killing subagents

Task close behavior depends on state:

- **Idle subagent:** closing removes the task row.
- **Active subagent:** closing asks for confirmation before killing the running agent.

This lets you clean up finished work while protecting active tasks from accidental termination.
