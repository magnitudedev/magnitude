## The lead agent

The lead agent is the agent you interact with directly. It coordinates your session, delegates to workers, and keeps your intent in focus across longer tasks.

The lead agent is designed to gather context and confirm its approach before making changes, keeping you in control. Before acting, it explores, communicates its assumptions, and checks in with you — so you can redirect early rather than undo later.

## The loop

A typical workflow looks like:

```
You send a message
  → Lead agent deploys an explorer to investigate
  → Lead agent discusses findings and proposes an approach
  → You confirm (or redirect)
  → Lead agent deploys builder(s) to implement
  → Lead agent deploys a reviewer to verify
  → Back to conversation
```

Not every task needs every step — the lead agent adapts based on complexity. Simple questions get answered directly. Quick fixes may skip exploration. Large changes get the full explore → plan → build → review treatment.

## Workers

The lead agent delegates to [six types of workers](/core-concepts/workers), each specialized for a different job. Multiple workers can run in parallel, each with its own context, tools, and model.

Workers report back when they finish. You can watch their progress in the task list.
