## The orchestrator

The orchestrator is the agent you interact with directly. For quick tasks, it can read files, run commands, and make small edits on its own. For anything substantial, it deploys **subagents** — specialized agents that run in parallel on isolated forks.

## The loop

A typical workflow looks like:

```
You send a message
  → Orchestrator deploys an explorer to investigate the codebase
  → Orchestrator discusses findings with you
  → Orchestrator deploys a planner
  → You confirm the approach
  → Orchestrator deploys a builder
  → Orchestrator deploys a reviewer to verify
  → Back to conversation
```

Not every task needs every step — the orchestrator adapts based on complexity. Simple questions get answered directly. Large changes get the full explore → plan → build → review treatment.

## Subagents

The orchestrator can deploy [6 types of subagents](/core-concepts/subagents), each specialized for a different job. Multiple subagents can run in parallel, and they share context through [artifacts](/core-concepts/artifacts).
