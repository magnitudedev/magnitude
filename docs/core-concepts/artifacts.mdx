Artifacts are named content stores that agents use to share context across work.

Common artifact types include:

- **Findings** (from explorers)
- **Plans** (from planners)

## Why artifacts matter

Artifacts let work compound across agents instead of being re-discovered every time.

For example:
- An explorer writes findings
- A planner reads those findings and writes a plan
- A builder uses the plan to implement
- A reviewer uses the same context to verify

## How artifacts appear in the UI

Artifacts are visible directly in the conversation:

- Inline chips like **[≡ plan]** or **[≡ report]**
- A clickable artifact side panel for browsing artifact contents

This makes handoffs visible and easy to inspect.

## Context passing between agents

When the orchestrator launches a subagent, it can pass artifacts in two modes:

- **context-artifact**: read-only context injected into the agent
- **writable-artifact**: read/write artifact the agent can create or update

This supports clean handoffs while controlling what each agent can modify.

## Storage

Artifacts are saved with your session data in `~/.magnitude/sessions/`.
