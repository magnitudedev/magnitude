Magnitude uses the [Agent Skills](https://agentskills.io) format for defining reusable agent behaviors. Skills are stored in `.magnitude/skills/` directories.

## What are skills?

A skill is a markdown file (`SKILL.md`) that defines a reusable behavior the agent can activate. Each skill lives in its own directory and contains instructions that modify how the agent approaches a task.

## Adding a skill

Create a `SKILL.md` file in one of the scan directories:

```
.magnitude/skills/my-skill/SKILL.md
```

### SKILL.md format

```markdown
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Instructions for the agent when this skill is active.

## Rules

1. Do this
2. Don't do that
3. Follow this pattern
```

The YAML frontmatter requires:
- **name** — skill identifier (used as the slash command)
- **description** — what the skill does

## Activating a skill

Type `/<skill-name>` in the chat to activate a skill. Skills show up alongside built-in commands in the autocomplete.

## Scan locations

Magnitude scans these directories for skills, in order of priority (later overrides earlier):

| Location | Scope |
|----------|-------|
| `~/.magnitude/skills/` | Global (available to all projects) |
| `<project>/.magnitude/skills/` | Project-local (overrides global) |

Project-local skills override global skills with the same name.

## First-run initialization

When you first run Magnitude and complete the setup wizard, built-in skills are automatically copied to `~/.magnitude/skills/`. These skills are available across all your projects.

You can create custom skills in either location:
- `~/.magnitude/skills/` — Global skills available to all projects
- `<project>/.magnitude/skills/` — Project-specific skills that override global ones

## Resetting skills

To reset global skills to their built-in defaults, run:

```bash
magnitude skills reset
```

Warning: This will overwrite all skills in `~/.magnitude/skills/`. You must type "yes" to confirm.

## Example: Debug skill

```markdown
---
name: debug
description: Take a scientific approach to debugging a problem
---

# Debug

You are debugging a difficult issue.
Approach with a scientific method — collect evidence, form hypotheses,
and verify them before declaring a root cause.

## Rules

1. Never jump to conclusions without a reproduction
2. Collect as much information as possible
3. Record evidence in `.magnitude/bugs/<date>/<title>.md`
4. Only pull the user in if absolutely necessary
```
