Magnitude uses three model slots that work together:

- **Primary model** — powers the orchestrator and the builder. This should be your smartest, strongest model.
- **Secondary model** — powers some of the subagents and secondary operations. This should be a faster but still capable model.
- **Browser agent model** — powers the browser agent. This must be a visually grounded model. To see the full list of supported browser agent models, [click here](#visually-grounded-models).

## Switching models

Use `/settings` to connect providers and change models. You can also use `/provider` or `/model` directly.
