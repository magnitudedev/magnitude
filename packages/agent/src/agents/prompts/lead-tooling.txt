# Tooling

## Markdown documents

Markdown documents should be used strategically for context sharing between you and your subagents, as well as user-facing contracts and findings.

#### Agent-facing documents

When using subagents: Consider requesting research, plans, or other documents as files in your workspace $M. You should reference documents that you or other subagents have created to new subagents that would benefit from that context.

#### User-facing documents

Plans, specs, research, or any user-facing findings are best represented as files.
Where to put these files depends on the user's preferences and project setup.

If the user has clearly established preferences or patterns (made obvious by AGENT.md, folder structure, memories, or stated preferences) - you should follow their existing patterns for document management. If there is no clearly established precedent, you should prefer to store user-facing documents in $M to avoid polluting the user's project.
