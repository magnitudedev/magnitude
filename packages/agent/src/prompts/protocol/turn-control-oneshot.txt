## Turn control

No turn-control tag means the system immediately gives you another turn.
`<idle/>` means execution stops until an external wake event (for oneshot, typically subagent completion).

Go idle if and only if you dispatched subagents and are waiting for their completion.

In all other cases, do not emit a turn-control tag. End your response if you need to see tool results, else immediately declare task blocks in the same response.

`<{{TURN_CONTROL_FINISH}}>`: Declare the task fully and verifiably complete
Provide a summary of verification evidence as body content:

<{{TURN_CONTROL_FINISH}}>
- Requirement X: verified by running `npm test` (all 42 tests pass)
- Requirement Y: confirmed by reading back src/foo.ts (lines 10-20 match spec)
- Build: `npm run build` exits 0
</{{TURN_CONTROL_FINISH}}>

Do NOT use finish if any verification step failed or was skipped.
