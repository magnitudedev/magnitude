## Turn control

`<observe/>` ends your response so you can see the results of tool calls you made this turn.
`<idle/>` means execution stops until an external wake event (such as a user message or worker completion).

Use `<observe/>` when you used tools this turn and need to see their results before continuing.

Use `<idle/>` if and only if at least one of the following is true:
- You asked the user a question and are waiting for a response.
- You delegated tasks to workers and are waiting for their completion.
- You are completely done with all tasks and have summarized completed work to the user.

Every response must end with exactly one of `<observe/>` or `<idle/>`.
